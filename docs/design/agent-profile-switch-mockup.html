改造前 → 改造后（对照）

原来： 允许作为 sub agent 仅展示徽章，必须点「编辑」进弹窗勾选 checkbox 才能改
现在： 卡片上一键切换，乐观更新 + PATCH 接口，失败回滚
  • 开关 = 启用/禁用该 Profile 作为 sub agent（对应字段 enabledAsSubagent）
  • 触发：乐观翻转 → PATCH /api/agent-profiles/:id { enabledAsSubagent } → 成功保留 / 失败回滚并提示
  • 内置 / 只读 Profile：开关锁定（后端 PATCH 会 403），标题弱化提示「未启用」
  • 无需新增依赖；复用与定时任务、MCP 卡片同一套约定式开关写法（符合设计语言：克制 / 一致）

需你确认 「启动 sub agent」的语义

  • 本方案按 「启用该 Profile 作为可被 AI 调用的 sub agent」（持续状态，适合开关）来设计。
  • 如果你指的是 「手动一键运行 / 测试某个 sub agent」（一次性动作），那应该是一个「运行」按钮 + 输入任务后触发，方向不同，请告诉我。
